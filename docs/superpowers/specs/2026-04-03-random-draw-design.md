# Random Draw (랜덤드로우) 사이트 — 설계 문서

## 개요

퀴즈 기반 랜덤 추첨 이벤트 사이트. 관리자가 퀴즈 이벤트를 생성하면 참여자들이 URL을 통해 접속하여 답변을 제출하고, 타이머 종료 후 관리자가 정답자 중에서 랜덤으로 당첨자를 추첨한다.

## 기술 스택

- **프론트엔드**: HTML/CSS/JS 단일 파일 (`index.html`)
- **데이터 저장**: Firebase Realtime Database (무료 플랜)
- **라우팅**: URL 해시 기반 (`#create`, `#event/{id}`, `#admin/{id}`)
- **배포**: 정적 호스팅 (GitHub Pages, Netlify 등)

## 화면 구성 (4개)

### 1. 메인 화면 (`#` 또는 `/`)

- 사이트 타이틀 + 간단한 설명
- "이벤트 생성하기" 버튼
- 심플하고 깔끔한 랜딩 페이지

### 2. 이벤트 생성 화면 (`#create`)

**입력 필드:**
- 관리자 닉네임 (필수)
- 관리자 비밀번호 (필수)
- 퀴즈 내용 — 질문 텍스트 (필수)
- 정답 — 보기 중 하나를 정답으로 지정 (필수)
- 보기 항목 — 최소 2개, 항목 추가/삭제 버튼 (필수)
- 타이머 선택 — 드롭다운 또는 버튼 그룹: 1분, 5분, 15분, 30분, 45분, 60분 (필수)

**동작:**
- "생성하기" 버튼 클릭 시 Firebase에 이벤트 데이터 저장
- 고유 이벤트 ID 생성 (Firebase push key)
- 생성 완료 후 공유 URL 표시 (`#event/{eventId}`)
- 관리자 접속 URL도 함께 표시 (`#admin/{eventId}`)
- 타이머는 이벤트 생성 시점부터 카운트다운 시작

### 3. 참여 화면 (`#event/{eventId}`)

**레이아웃: 카드 중심 세로형**
- 상단 중앙: 원형 카운트다운 타이머
- 퀴즈 카드: 질문 텍스트 표시
- 닉네임 입력 필드
- 보기 목록: 세로 리스트, 라디오 버튼 스타일 (선택 시 하이라이트)
- "제출하기" 버튼

**동작:**
- 타이머가 실시간으로 카운트다운
- 타이머 종료 시 제출 버튼 비활성화 + 안내 메시지
- 제출 시 Firebase에 `{닉네임, 선택한 답, 제출시간}` 저장
- 제출 완료 후 "제출 완료" 대기 화면 표시
- 동일 닉네임 중복 제출 방지

### 4. 관리자 추첨 화면 (`#admin/{eventId}`)

**진입:**
- 닉네임 + 비밀번호 입력 폼
- 이벤트 생성 시 설정한 값과 일치해야 접속 가능

**레이아웃: 좌우 분할**

왼쪽 메인 영역 (상단에서 하단 순서):
1. **추첨 결과 영역** — 추첨된 닉네임을 크게 표시, 몇 번째 당첨자인지 표시
2. **추첨하기 버튼** — 그라데이션 스타일, 클릭 시 랜덤 추첨 실행
3. **통계 카드** — 전체 참여자 수 / 정답자 수 / 남은 추첨 대상 수
4. **퀴즈 정보** — 퀴즈 내용 + 정답 표시

오른쪽 당첨자 패널:
- "당첨자 목록" 헤더
- 당첨 순번 + 닉네임 리스트
- 최신 당첨자 하이라이트 (테두리 + 색상)
- 총 당첨자 수 표시

**추첨 로직:**
- 정답자 목록에서 이미 당첨된 사람을 제외하고 랜덤 선택
- 추첨 시 슬롯머신 스타일 애니메이션 효과
- 더 이상 추첨할 정답자가 없으면 버튼 비활성화 + 안내

## 데이터 구조 (Firebase Realtime DB)

```
events/
  {eventId}/
    admin/
      nickname: string
      password: string (해시 권장, 단 단일 HTML이므로 평문 허용)
    quiz/
      question: string
      answer: string (정답 보기의 값)
      options: string[] (보기 목록)
    timer/
      duration: number (분 단위)
      startedAt: number (timestamp)
    responses/
      {responseId}/
        nickname: string
        answer: string
        submittedAt: number (timestamp)
    winners/
      {index}/
        nickname: string
        drawnAt: number (timestamp)
```

## URL 라우팅

| 해시 | 화면 | 설명 |
|------|------|------|
| `#` 또는 없음 | 메인 | 랜딩 페이지 |
| `#create` | 이벤트 생성 | 관리자가 이벤트 설정 |
| `#event/{id}` | 참여 | 사용자가 퀴즈에 답변 |
| `#admin/{id}` | 관리자 추첨 | 정답자 중 랜덤 추첨 |

## 주요 동작 흐름

1. 관리자가 메인 화면에서 "이벤트 생성하기" 클릭
2. 이벤트 생성 화면에서 정보 입력 후 "생성하기"
3. Firebase에 이벤트 저장, 고유 URL 생성
4. 관리자가 참여 URL을 사용자들에게 공유
5. 사용자들이 URL로 접속, 닉네임 입력 + 정답 선택 + 제출
6. 타이머 종료 시 더 이상 제출 불가
7. 관리자가 관리자 URL로 접속, 닉네임/비밀번호 입력
8. "추첨하기" 버튼을 눌러 정답자 중 랜덤 추첨
9. 당첨자는 오른쪽 패널에 누적 표시
10. 필요한 만큼 반복 추첨 (이미 당첨된 사람은 제외)

## 디자인 톤

- 다크 테마 (배경 #0f172a 계열)
- 포인트 컬러: 파랑(#4a9eff), 초록(#10b981), 노랑(#fbbf24), 빨강(#ef4444)
- 둥근 카드 스타일 (border-radius: 10-12px)
- 깔끔하고 모던한 UI

## 예상 규모

- 동시 참여자: 10~100명
- Firebase 무료 플랜 (Spark)으로 충분
- 이벤트 데이터는 일회성 — 종료 후 보관 불필요

## 제약사항 및 고려사항

- 단일 HTML 파일로 모든 화면 포함 (CSS/JS 인라인)
- Firebase SDK는 CDN으로 로드
- Firebase 프로젝트 설정(apiKey, databaseURL 등)은 HTML 내 인라인으로 하드코딩 (사용자가 자신의 Firebase 프로젝트 정보로 교체)
- 비밀번호는 클라이언트 사이드에서 비교 (보안 수준 낮음, 이벤트 용도에 적합)
- 모바일 반응형 지원 필요 — 관리자 추첨 화면의 좌우 분할은 모바일에서 상하 스택으로 전환
- 추첨 애니메이션은 CSS 기반으로 구현
- 중복 닉네임 제출 방지는 클라이언트 사이드에서 Firebase 조회 후 체크
- 타이머는 이벤트 생성 시점부터 시작 — 관리자가 URL을 공유하는 시간도 타이머에 포함됨을 인지
